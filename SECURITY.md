# Security Policy

## Supported versions

Only the latest published version of the Pine Script VS Code extension receives security fixes.

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |
| Older   | ❌        |

## Reporting a vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

If you believe you have found a security issue in the extension, please report it responsibly by following these steps:

1. **Email** the maintainer directly at the address listed on the [VS Code Marketplace extension page](https://marketplace.visualstudio.com/search?term=pinescript&target=VSCode).
2. Include as much detail as possible:
   - A description of the vulnerability and its potential impact.
   - Steps to reproduce or a proof-of-concept (if applicable).
   - The extension version and VS Code version you are using.
3. You will receive an acknowledgement within **5 business days**.
4. We aim to provide a fix or mitigation plan within **30 days** of confirmation.

We appreciate responsible disclosure and will credit researchers (with their permission) in the release notes once a fix is published.

## Scope

This policy covers only the Pine Script VS Code extension distributed through the VS Code Marketplace. It does not cover third-party dependencies, TradingView services, or any other external systems.
